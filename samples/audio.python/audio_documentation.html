Gestalt Audio Player Widget

To use the Audio Widget you will need to have Silverlight installed. You can get Silverlight at: http://www.silverlight.net.

The Audio Gestalt Widget has been designed to support a simple mode where you only only will need to add a reference to two JavaScript files plus some HTML5 audio tag markup and you are good to go. To see an example of this this take a look at the embed sample

If you would like to modify the user interface appearance you can change it in two ways:

  1. Simple adjustment of the basic colors can be done by modifying color values in the source code:
    ButtonOffHexValue = “#ffbbbbbb” # color of buttons when mouse is not over
    ButtonOverHexValue = “#ffeeeeee” # color of buttons when mouse is hovering
    PanelBackgroundHexValue = “#44ffffff” # The control panel background color
    TextColorHexValue = “#ff808080” # the color of the timecode and caption
    MediaBackDropHexValue = “#ff000000” # the color of the overall media player background

  2. If you want to dramatically modify the UI, you will need to change the XAML that defines it in the added skin attribute of the audio tag. You can either manually do this in a text editor, or you can load up the original XAML file in Expression Blend 3. Just save the changes to a new file named "newskin.xaml" that's in the same directory as the original file modify the "skin" attribute as shown below, and refresh the browser to view the changes. Like so:

    <audio id="audio1"
    ..
    skin="newskin.xaml">
    <source src="path to file"></source>
    </audio>

There are two skins provided as examples: media.xaml and media_alternative.xaml

Attributes supported by the Audio Widget:

  • id [required] - the audio tag must have a unique id defined
  • autoplay - When set to true specifies that the media should play immediately. The default value is true
  • volume - A decimal value ranging between 0 and 1. The default value is .5
  • width - Defines the width of the player in pixels. The default value is 300
  • height - Defines the height of the player in pixels. The default value is 150.
  • loop - Specifies to repeat the content when done playing the file or set of files. The default value is false
  • skin - This is the XAML that will be loaded up for the player UI. The default value is media.xaml
